## 本轮已确认需求
- 独立项目 + 独立端口 + 独立命名空间 `openclaw-webchat`
- 每个 agent 保留一条长期主时间线；`/new` 仅重置上游上下文，并在历史中显示分隔标记
- 历史只保留 `openclaw-webchat` 渠道消息，不混默认 WebUI 和其他渠道
- 支持 assistant 富媒体渲染（图片/音频/视频/文件）
- 富媒体内部协议优先采用结构化 attachment / media block，兼容 `mediaUrl:` / `MEDIA:` fallback
- 支持用户上传图片、音频；音频默认转写后发送，同时保留原始文件引用
- 输入区使用单一 `+` 按钮统一上传图片 / 音频
- 图片直接显示在消息气泡中，支持单击查看、缩放和平移
- 视频直接显示在消息气泡中，使用原生控件处理播放与全屏
- 对话气泡支持标题、列表、引用、代码块、链接等常用 Markdown 渲染
- assistant 回复中的 Markdown 图片语法 `![alt](url)` 会被提取为图片块，并保持与正文的原始交错顺序
- 消息解析与渲染顺序由共享模块 `public/message-blocks.js` 统一维护
- 当前版本中，图片 / 视频图文气泡宽度由媒体实际显示宽度决定，桌面端最大到 `70vw`
- 输入框右侧提供 `/` slash 菜单按钮；至少支持 `/new`、`/model`、`/think`，并补齐 `/fast`、`/verbose`、`/compact`、`/help`
- slash 命令采用本地执行语义：白名单命令走本地 `/command` 接口，未知 slash 继续按普通消息发送给 agent
- 设置面板支持统一管理用户自己和所有 agent 的显示名 / 头像
- 头像支持本地图片选择、正方形裁剪与上传保存
- 设置面板采用可收起的展开式结构，可继续扩展语种、主题等偏好设置
- 发送后会立即清空输入框，并在等待阶段显示 agent 头像与动态处理中提示
- 左右聊天头像尺寸一致，左栏列表头更简化
- 左侧 agent 列表：头像跨两行 + 名字 + 最后一条摘要 + 活跃状态
- 提供 agent 个性化显示设置（仅影响本项目页面显示）
- 手机端抽屉布局，平板保留左右布局
- 右侧消息区以输入框为原点贴底显示；新消息与媒体加载完成后自动滚到最新位置
- 左栏与右栏滚动彼此独立
- 当前移动端测试分支中，首次进入会话只预载入 `15` 条历史，后续历史分页仍保持单页 `30` 条
