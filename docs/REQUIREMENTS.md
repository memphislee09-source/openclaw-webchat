# Requirements — openclaw-webchat

## 项目定位
`openclaw-webchat` 是一个独立于 OpenClaw 默认 WebUI 的 WebChat 项目，具备独立端口、独立会话命名空间、独立历史存储和独立前端实现。

## 核心目标
1. **历史保留**：通过 `openclaw-webchat` 与 agent 的历史对话长期保留，并支持类似微信的无限上拉加载。
2. **富媒体增强**：agent 能向用户发送图片、音频、视频、文件，页面可正确渲染；媒体失效时显示“文件丢失”。

## 产品边界
- 独立项目 + 独立端口。
- 与默认 WebUI、Slack、Discord 等其他渠道会话完全隔离。
- 访问方式以本机/内网为主，外网访问通过 Tailscale。
- MVP 不做额外登录鉴权。

## 会话模型
- 左侧为 agents 列表。
- 点击 agent：
  - 若存在 `openclaw-webchat` 渠道会话，则直接进入；
  - 否则自动创建。
- 每个 agent 在 `openclaw-webchat` 下维护一条长期保留的主时间线。
- `/new` 仅重置上游上下文，不清除本地历史。
- 历史中需插入“已重置上下文”分隔标记。
- agent 不必记住旧历史，但用户可在界面持续查看旧历史。

## 消息展示
- 仅展示：用户消息、assistant 最终回复。
- 不展示：工具调用、中间执行代码阶段、reasoning、调试过程。
- 存储层也仅保留可展示消息。

## 富媒体能力
### Agent → User
- 支持：图片、音频、视频、文件。
- 优先正确渲染；资源丢失时显示“文件丢失”。
- 协议双兼容：
  - 兼容 `openclaw-agent-chat-ui` 现有 media 协议
  - 兼容未来 OpenClaw 原生结构化媒体返回

### User → Agent
- 支持上传：图片、音频。
- 不支持上传：视频。
- 图片默认用于“让 agent 理解图片”，同时保留原始文件引用。
- 音频默认自动转文字再发，同时保留原始文件引用。
- 若音频转写失败，仍发送原始音频，并提示“转写失败”。
- 后续允许用户通过提示词要求 agent 将上传图片/音频作为文件保存，而非仅做内容识别。

## 左侧列表与设置
- 左侧每个 agent 条目布局：
  - 头像固定最左，跨两行；
  - 右侧第一行：agent 名字；
  - 右侧第二行：最后一条会话摘要；
  - 超长摘要省略显示。
- 活跃状态：
  - 正在回复时为活跃；
  - 最近有回复也需体现弱状态感知。
- 支持 agent 个性化设置：
  - 自定义显示名；
  - 自定义头像；
  - 仅影响 `openclaw-webchat` 页面显示。
- 左侧栏底部提供“设置”入口，后续可扩展更多设置功能。

## 摘要显示规则
- 纯图片：`[图片]`
- 纯音频：`[音频]`
- 纯视频：`[视频]`
- 纯文件：`[文件]`
- `/new` 分隔：`已重置上下文`
- 文本 + 媒体：优先显示文本，无文本时显示媒体标签

## 交互体验
- assistant 最终回复：文本流式显示，媒体在最终完成后补上。
- 平板：保留左右布局。
- 手机：左侧 agents 列表收为侧滑抽屉。

## 工程约束
1. 尽量降低对 OpenClaw 内部前端实现的耦合，避免每次升级都需大量适配。
2. 借鉴 OpenClaw 官方前端的布局和风格，但不深度绑定其内部结构。
3. 富媒体和历史能力优先复用/借鉴 `openclaw-agent-chat-ui` 的成熟方案。
4. 搜索/筛选历史不在 MVP，放入第二阶段。
