# Project Charter — openclaw-webchat

## 1. 愿景
构建一个独立、稳定、可长期演进的 OpenClaw WebChat 产品，为 agent 对话提供更强的历史连续性和富媒体体验。

## 2. MVP 目标
- 提供独立于默认 WebUI 的聊天入口。
- 左侧展示 agents 列表，点击即恢复/创建该 agent 的 `openclaw-webchat` 会话。
- 保留每个 agent 的长期主时间线，支持无限上拉加载历史。
- 支持 assistant 富媒体输出渲染（图片/音频/视频/文件）。
- 支持用户上传图片和音频。
- 对话界面仅展示最终可读消息，不展示工具执行过程。
- 支持移动端和平板访问。

## 3. 非目标（MVP 不做）
- 与默认 WebUI 的历史互通。
- 与 Slack、Discord 等其他渠道互通历史。
- 历史搜索、日期筛选、跳转定位。
- 额外登录鉴权系统（通过 Tailscale 访问）。
- 用户上传视频。

## 4. 成功标准
- 任一 agent 均可一键进入专属聊天页。
- `/new` 后历史仍可完整回溯，并显示分隔标记。
- 上拉可持续加载更早历史，体验接近微信式聊天记录回溯。
- assistant 富媒体在页面中可稳定渲染；失效媒体有清晰兜底。
- OpenClaw 升级后，项目仅需在适配层小范围处理，不依赖官方前端内部细节重构。
- 在手机/平板/桌面上都可正常使用。

## 5. 关键约束
- 对 OpenClaw 官方前端保持“借鉴 UI，不深度耦合代码”的策略。
- 历史与富媒体链路优先自主管控。
- 系统层面保持独立命名空间、独立存储、独立接口。
- 存储层仅保留可展示消息，避免中间执行日志污染历史。
