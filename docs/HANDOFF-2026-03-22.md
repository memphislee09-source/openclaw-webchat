# Handoff — 2026-03-22

## 当前结论
`openclaw-webchat` 仍应继续以 `main` 为唯一开发基线。2026-03-22 这一轮已把历史搜索第二阶段的首批增强、自测补强、对 gateway CLI 混合 stdout 日志的兼容解析、当前 agent 级模型切换弹窗、最近一批聊天区细节修复、输入框发送/停止双态按钮和 gateway `chat.abort` 停止链路，以及公开发布准备第一批资料补到主线工作树，并已在本地实例上完成 `SELFTEST_OK` 验证与 release bundle 打包验证。

当前已确认的主线状态：
- 当前版本仍为 `0.1.4`
- 默认服务地址仍为 `http://127.0.0.1:3770`
- 当前开发基点仍为 `main`
- 用户级 LaunchAgent `ai.openclaw.webchat` 继续作为常驻运行方式
- 当前主线新增的历史搜索第二阶段首批能力：
  - 支持当前 agent 主时间线的日期筛选（起始/结束日期）
  - 支持更大结果集切换，前端可选 `20 / 50 / 100`
  - 搜索匹配从简单整句包含增强为分词全命中、紧凑匹配与分数排序
  - 搜索命中高亮同步增强为整句 + 分词高亮
- 当前主线新增的上游兼容修正：
  - `gateway call` 如果 stdout 前缀混入插件诊断日志，WebChat 仍会从混合输出里提取真实 JSON
  - 因此 `/model`、`/think` 等依赖 `models.list` / `sessions.list` 的本地 slash 命令已恢复稳定
- 当前主线新增的模型切换能力：
  - 在当前 agent 中输入无参 `/model` 或 `/models`，前端会弹出模型切换弹窗，而不再向消息流写入纯文本提示
  - 弹窗顶部显示当前 agent 上游 session 正在使用的模型
  - 弹窗列表展示当前可用模型，格式为 `provider/model`，例如 `openai-codex/gpt-5.4`
  - 用户点击某个模型后，会立即把该 agent 绑定的上游 session 切换到所选模型
  - 服务端已补 session 级 `model-options` / `model` 接口，前端与自测统一复用这套数据源
- 当前主线新增的聊天区细节修复：
  - 消息区头像尺寸已与左栏 agent 头像统一，不再出现同一联系人在侧栏和消息区大小不一致的问题
  - 用户发言后，当前会话在 agent 处理中阶段会继续保持贴底，不再短暂下沉并看起来像停留在上一条会话
  - 视频消息在未播放前会显示可见预览层，优先展示首帧；即使拿不到首帧，也至少显示视频标题和播放提示
  - 输入框里的“发送”文本按钮已改成图标按钮；当前 agent 处理中时会自动切换成停止图标
  - 用户点击停止图标后，会直接停止当前 agent 绑定上游 session 的当前任务，而不是只能等待回复自然结束
- 当前主线新增的停止链路：
  - 服务端已补 `POST /api/openclaw-webchat/sessions/:sessionKey/stop`
  - 该接口会调用 gateway `chat.abort` 停止当前 upstream session 的 active run
  - 前端停止后会立即退出当前忙碌态，并阻止“迟到回复自动补回”继续写入当前时间线
  - 对“附件仍在准备、用户已点击停止”的前端场景也已做兜底，避免用户明明点了停止却继续把消息发出去
- 当前主线新增的公开发布资料：
  - README 首页已按公开仓库入口重写，补上截图、安装方式入口、公开发布 checklist 与安全边界说明
  - `docs/PUBLIC_RELEASE_CHECKLIST.md` 已补齐公开推荐前的产品、文档、发布与社区投放检查项
  - 已新增两套给 OpenClaw agent 使用的安装说明：
    - `docs/AGENT_INSTALL_BUNDLE.md`
    - `docs/AGENT_INSTALL_NETWORK.md`
  - 两套说明都按“逐步执行 + 每步检查 + 失败即停”的方式编写，并额外考虑了低能力模型的执行策略
  - 已新增 `scripts/build-release-bundle.sh`，可生成 release bundle 与 SHA256 校验文件
  - 已归档当前 UI 截图到 `docs/images/`，可直接用于 README 和 GitHub Release 页面
- 当前环境侧修正：
  - `memory-lancedb-pro` 已补安装来源记录，OpenClaw 不再把它视作无 provenance 的未追踪本地插件
