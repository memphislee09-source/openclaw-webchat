## 已实现能力
- 独立项目 + 独立端口 + 独立命名空间 `openclaw-webchat`
- `agentId -> session` 长期绑定
- 自有 JSONL 历史存储，仅保留可展示消息
- `/new` 仅重置上游上下文，本地历史保留，并写 marker `已重置上下文`
- 隐藏 bootstrap 注入，建立类一等渠道行为
- assistant 最终回复抓取
- 结构化媒体块优先，兼容 `mediaUrl:` / `MEDIA:` fallback
- 本地媒体代理与“文件丢失”兜底
- 左侧 agent 列表 + 右侧聊天主区 + 输入区
- 历史无限上拉分页
- 手机端抽屉布局
- 底部起消息流（最新消息贴近输入框）
- 对话气泡头像（agent 左侧、用户右侧）
- 新消息与新媒体内容自动贴底显示，无需手动滚动
- 左栏与右栏独立滚动
- 左下角设置入口，支持“我的显示名 / 我的头像 URL”快速保存
- 用户图片上传：发送前预览、服务端落盘、与文本同发
- 用户音频上传：原音频保留、本地 Whisper 默认自动转写、失败时仍发送原音频
- 输入区单一 `+` 按钮统一上传图片 / 音频
- 可视化设置页：改为统一联系人编辑器，支持用户自己和所有 agent 共用一套显示设置流
- assistant 媒体卡片细化：音频转写提示、媒体加载失败兜底、文件元信息展示
- 图片直接嵌入气泡：支持单击全屏、缩放与拖拽平移
- 视频直接嵌入气泡：取消双击全屏，改为使用原生播放器控件
- 头像支持本地图片选择、前端正方形裁剪和上传保存
- 设置面板改为可收起的展开式结构，已预留“偏好”等后续扩展位
- 用户发送后立即清空输入框；等待期间显示 agent 头像 + 动态处理中指示
- 左右聊天头像尺寸统一；左栏列表头已简化
- gateway 调用断连重试
- 长耗时媒体回复等待窗口提升到 120s（默认）
