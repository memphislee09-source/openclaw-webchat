# Roadmap — openclaw-webchat

## 发布基线
- [x] `0.1.1` 基线冻结（manifest 版本 `0.1.1`）
- [x] `0.1.2` 合入 `main`，作为后续开发基点
- [x] `0.1.3` 合入 `main`，补齐历史搜索第一版与交互收口
- [x] `0.1.4` 合入 `main`，补齐公开化文档、访问与安全设置与中英文界面切换

## Milestone 0：需求冻结与方案确认
- [x] 建立仓库与基础文档
- [x] 明确 MVP 范围
- [x] 冻结核心需求与约束
- [x] 确认独立项目/独立端口策略
- [x] 确认当前技术实施方案（Node/Express 单体 + 原生前端 + OpenClaw 适配层）

## Milestone 1：服务端最小闭环
- [x] 建立 `openclaw-webchat` 独立命名空间与路由
- [x] 完成 `agentId -> session` 绑定逻辑
- [x] 建立自有历史存储（JSONL）
- [x] 支持 `/new` 分隔标记写入
- [x] 完成消息净化（仅保留可展示消息）

## Milestone 2：前端 MVP
- [x] 左侧 agent 列表（头像跨两行 + 名字 + 摘要 + 状态）
- [x] 点击 agent 自动恢复/创建会话
- [x] 聊天消息区 + 输入区
- [x] 文本流式显示（当前为最终回复直出，后续可继续增强流式细节）
- [x] 无限上拉加载历史
- [x] 移动端抽屉布局

## Milestone 3：富媒体与上传
- [x] assistant 图片/音频/视频/文件渲染（基础链路已通）
- [x] 媒体失效兜底
- [x] 用户上传图片
- [x] 用户上传音频 + 自动转写
- [x] 输入区统一 `+` 上传入口（图片 / 音频）
- [x] 图片直接嵌入气泡，支持查看器缩放与平移
- [x] 视频直接嵌入气泡，回归原生播放器全屏
- [x] 对话气泡支持常用 Markdown 渲染
- [x] 媒体协议双兼容
- [x] 实验分支验证视觉媒体气泡等宽方案（图片 / 视频图文气泡按实际显示宽度，桌面端最大 `70vw`）

## Milestone 4：设置与稳定性
- [x] agent 个性化显示设置（显示名/头像）
- [x] 左侧底部设置入口
- [x] 我的头像/显示名快速设置（URL 方式）
- [x] 可视化设置页（统一联系人管理 + 展开式结构）
- [x] 头像本地裁剪上传
- [x] 消息区贴底渲染与新消息自动滚动
- [x] 发送后立即清空输入框与处理中动态指示
- [x] 设置面板可收起展开与头像尺寸统一
- [x] 左右栏独立滚动
- [x] 回归测试与升级兼容性检查（基础自测 + baichai 发图验证 + settings/upload 接口补测 + 统一上传入口骨架检查）
- [x] 自测完成，准备验收

## Milestone 5：`0.1.3` 主线收口（当前）
